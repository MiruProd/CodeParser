# CodeParser

Удобный графический инструмент (GUI) на PyQt6 для подготовки и упаковки исходного кода проектов в единый текстовый файл контекста. Утилита предназначена для быстрой передачи кодовой базы в большие языковые модели (LLM / нейросети, такие как ChatGPT, Claude, Gemini). 

CodeParser автоматически отсекает бинарные данные, системные директории, лок-файлы и позволяет гибко выбрать только те файлы, которые необходимы для решения вашей задачи.

---

## 🚀 Возможности

*   **Интерактивное дерево файлов:** Наглядный выбор директорий и отдельных файлов с помощью чекбоксов. Автоматический расчет общего размера выбранных файлов в реальном времени.
*   **Оценка токенов:** Приблизительный расчет количества токенов для выбранного объема данных (с учетом кодировок и синтаксиса).
*   **Умная фильтрация:**
    *   Поддержка локальных правил `.gitignore` для автоматического скрытия служебных файлов.
    *   Исключение бинарных форматов данных (изображения, архивы, исполняемые файлы, шрифты).
    *   Игнорирование тяжелых лок-файлов и автогенерации (`package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`, `go.sum`, файлы локализации и лицензии).
*   **Стековые пресеты расширений:** Быстрая фильтрация по типам файлов для популярных стеков разработки (Python, Go/Wails/Svelte, Web/Frontend, Rust, C/C++, Java/Kotlin).
*   **Генерация структуры:** Построение чистого ASCII-дерева проекта, включающего только выбранные для экспорта элементы.
*   **Динамические темы оформления:** Быстрое переключение между Темной (в стиле VS Code) и Светлой темами.
*   **Два режима вывода:** Прямое копирование сформированного контекста в буфер обмена или сохранение в физический `.txt` файл на диске.

## 📂 Пример структуры выходного файла

Результат работы утилиты генерируется в чистом текстовом формате, который отлично распознается LLM:

```text
=== ПОЛНАЯ СТРУКТУРА ПРОЕКТА (БЕЗ СИСТЕМНОГО МУСОРА) ===

MyProject/
├── src/
│   ├── core/
│   │   └── parser.py
│   └── main.py
└── README.md

================================================================================

=== СОДЕРЖИМОЕ КЛЮЧЕВЫХ ФАЙЛОВ КОДА ===

<file path="src/main.py">
# Содержимое файла main.py...
</file>

<file path="src/core/parser.py">
# Содержимое файла parser.py...
</file>
```

## 🛠 Архитектура проекта

Кодовая база разделена на независимые слои для упрощения поддержки:
*   `src/core/` — чистая бизнес-логика: обход файловой системы, парсинг `.gitignore`, фильтрация и генерация финального текста. Этот слой полностью изолирован от графической библиотеки.
*   `src/ui/` — интерфейс приложения: описание виджетов, стилизация (QSS-шаблоны) и обработка пользовательских событий.
*   `build_scripts/` — скрипты для автоматизации локальной компиляции проекта.
*   `.github/workflows/` — сценарии CI/CD для облачной сборки релизов в GitHub Actions.

## 🔧 Установка и запуск (Разработка)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ВАШ_ПРОФИЛЬ/CodeParser.git
   cd CodeParser
   ```

2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv .venv
   # Для Windows:
   .venv\Scripts\activate
   # Для macOS / Linux:
   source .venv/bin/activate
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Запустите приложение:
   ```bash
   python src/main.py
   ```

## 📦 Сборка исполняемого файла (.exe)

### Локальная сборка
Вы можете скомпилировать проект в один автономный исполняемый файл для Windows, не требующий наличия установленного Python:
```bash
pip install pyinstaller
python build_scripts/build_exe.py
```
Готовый файл появится в директории `dist/CodeParser.exe`.

### Облачная сборка через GitHub Actions
В репозитории настроена автоматическая сборка релизов. При публикации нового тега версии GitHub автоматически создаст релиз и прикрепит к нему скомпилированный файл:
```bash
git tag v1.0.0
git push origin v1.0.0
```
