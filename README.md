# CodeParser — NextWG Smart Code Packer

Инструмент для подготовки и упаковки исходного кода проектов в единый текстовый файл контекста для последующей передачи в большие языковые модели (LLM / нейросети). Утилита помогает быстро отсечь бинарные файлы, системные директории и собрать только те файлы кода, которые необходимы для решения конкретной задачи.

## Возможности

*   **Интерактивный выбор:** Графический интерфейс на PyQt6 с интерактивным деревом каталогов и чекбоксами для точечного выбора файлов.
*   **Гибкая фильтрация:** Парсинг локального файла `.gitignore`, автоматическое исключение лок-файлов (`yarn.lock`, `package-lock.json`, `go.sum`) и бинарных форматов данных.
*   **Стековые пресеты:** Встроенные наборы расширений под популярные направления разработки (Python, Go/Svelte, Rust, Web/Frontend, C++, Java/Kotlin).
*   **Генерация структуры:** Построение чистого ASCII-дерева проекта, адаптированного только под выбранные вами для экспорта файлы.
*   **Динамические темы:** Поддержка темной (VS Code-style) и светлой тем оформления на основе переиспользуемых цветовых палитр Python.
*   **Оценка объема:** Расчет суммарного объема выбранных файлов в килобайтах и приблизительный расчет количества токенов.
*   **Автоматизация релиза:** Настроенный CI/CD-процесс для сборки EXE-файла на стороне GitHub при публикации тега версии.

## Архитектура проекта

Код разделен на независимые слои для упрощения поддержки и масштабирования:
*   `src/core/` — бизнес-логика приложения: правила игнорирования, сканирование диска, константы и генерация итогового текста. Данный слой полностью изолирован от графического интерфейса.
*   `src/ui/` — презентационный слой: компоновка виджетов, цветовые палитры, QSS-темы и обработчики сигналов UI.
*   `build_scripts/` — автоматизация компиляции проекта на локальном компьютере.
*   `.github/workflows/` — сценарии облачной сборки релиза в среде GitHub Actions.

## Установка и запуск (Разработка)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ВАШ_ПРОФИЛЬ/CodeParser.git
   cd CodeParser
   ```

2. Настройте виртуальное окружение и установите зависимости:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # На Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. Запустите приложение:
   ```bash
   python src/main.py
   ```

## Сборка исполняемого файла (.exe)

### Локальная компиляция
Вы можете собрать автономное Windows-приложение на своем локальном компьютере с помощью скрипта автоматизации:
```bash
pip install pyinstaller
python build_scripts/build_exe.py
```
Готовый исполняемый файл будет сохранен в папку `dist/CodeParser.exe` в корне проекта.

### Облачная сборка через GitHub Actions
В проекте настроен автоматический рабочий процесс. Для создания нового релиза в вашем репозитории на GitHub достаточно назначить и отправить тег версии:
```bash
git tag v1.0.0
git push origin v1.0.0
```
Служба GitHub Actions сама запустит виртуальную машину Windows, скомпилирует проект и прикрепит файл `CodeParser.exe` к созданному релизу.
